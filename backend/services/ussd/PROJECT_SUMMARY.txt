🚀 QUICK START GUIDE
═══════════════════════════════════════════════════════════════════════════════

1. INSTALL DEPENDENCIES
   npm install

2. SET ENVIRONMENT VARIABLES
   Copy config.example.js and set your API keys:
   - AFRICAS_TALKING_API_KEY

   redis-server

5. TEST THE SYSTEM
   # See QUICK_REFERENCE.js for curl examples
   curl http://localhost:3001/health

6. MONITOR
   # Get active sessions
   curl http://localhost:3001/stats

⚠️ IMPORTANT NOTES
═══════════════════════════════════════════════════════════════════════════════

1. Redis Required
   └─ Must have Redis 6+ running on localhost:6379 (or configured URL)

2. API Keys Required
   ├─ Africa's Talking (USSD provider)
   ├─ Cell Tower API (location)
   ├─ Google Maps or OpenCage (reverse geocoding)
   └─ SMS Gateway (fallback)

3. Network Code (MCC/MNC)
   └─ Required from USSD provider for cell tower location
     Example: 63902 = Safaricom Kenya

4. Session Timeout
   └─ Default 10 minutes - adjust config.sessionTimeout if needed

5. SMS Fallback
   └─ Auto-triggers after 2 USSD timeouts
   └─ Adjust config.smsConfig.maxUSSDRetries

6. Rate Limiting
   └─ Implement per-phone limiting in production
   └─ See config.security.rateLimiting
